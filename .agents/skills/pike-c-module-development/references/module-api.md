# Pike C Module API Reference

## .cmod File Structure

A `.cmod` file is preprocessed by Pike's build system into a `.c` file. It contains
standard C includes and declarations followed by Pike-specific macros.

### Skeleton

```c
/* -*- c -*- */

#include "global.h"
#include "interpret.h"
#include "svalue.h"
#include "stralloc.h"
#include "array.h"
#include "mapping.h"
#include "object.h"
#include "program.h"
#include "pike_types.h"
#include "pike_error.h"
#include "builtin_functions.h"
#include "module_support.h"

DECLARATIONS

/*! @module MyModule
 *! Description of the module.
 */

PIKECLASS MyClass
{
  CVAR struct my_storage storage;

  PIKEFUN void create()
  {
    /* constructor body */
  }

  INIT { /* C-level init */ }
  EXIT { /* C-level destructor */ }
}

PIKE_MODULE_INIT { /* module init */ }
PIKE_MODULE_EXIT { /* module cleanup */ }

/*! @endmodule
 */
```

## Class and Function Macros

### PIKECLASS(Name) { ... }

Defines a Pike class. The body contains CVAR, PIKEFUN, PIKEVAR, INIT, EXIT, EXTRA, and
optional C helper functions.

```c
PIKECLASS MyClass
{
  /* members and methods */
}
```

### PIKEFUN return_type name(args) { ... }

Defines a Pike-callable function within a PIKECLASS or at module level.

The `return_type` uses Pike type syntax: `void`, `int`, `string`, `float`, `array`,
`mapping`, `multiset`, `object`, `program`, `mixed`, `function`, or `array(type)`.

Arguments are declared with Pike types. The precompiler generates type checking code.

```c
PIKEFUN int add(int a, int b)
{
  RETURN a + b;
}

PIKEFUN string format(string fmt, mixed ... args)
{
  f_sprintf(args + 1);
  RETURN Pike_sp[-1].u.string;
}

PIKEFUN void do_something()
{
  pop_n_elems(args);
}
```

### PIKEFUN Flags

Flags modify function visibility and behavior. Applied after the function signature.

| Flag | Value | Effect |
|------|-------|--------|
| `ID_STATIC` | 0x01 | Not visible by indexing (legacy alias for `ID_PROTECTED`; prefer `ID_PROTECTED`) |
| `ID_PROTECTED` | 0x01 | Not visible by indexing |
| `ID_PRIVATE` | 0x02 | Not visible by inherit |
| `ID_FINAL` | 0x04 | May not be overloaded |
| `ID_PUBLIC` | 0x08 | Anti-private (visible despite private inherit) |
| `ID_OPTIONAL` | 0x100 | Not required by the interface |

```c
PIKEFUN void internal_method()
  flags ID_PROTECTED;
{
  /* only callable from within Pike and inheriting classes */
}

PIKEFUN string to_string()
  flags ID_PUBLIC;
{
  /* visible even through private inherit */
}
```

### PIKEVAR

Declares a Pike-visible instance variable within a PIKECLASS. The precompiler
generates getter/setter infrastructure and handles reference counting.

```c
PIKEVAR function(int, int, string:void) callback
  flags ID_PRIVATE;
```

## Storage Macros

### CVAR type name;

Declares per-object C-level storage. Access via the `THIS` macro.

```c
CVAR struct {
  int fd;
  char *buffer;
  size_t buf_len;
} storage;

CVAR struct string_builder buf;
```

### ADD_STORAGE(struct_type)

Registers the size of per-object C storage. Called within `PIKE_MODULE_INIT`
when not using PIKECLASS (for raw program creation).

```c
PIKE_MODULE_INIT
{
  ADD_STORAGE(struct my_storage);
}
```

## Object Access

### THIS

Macro expanding to `((struct storage_type *)(Pike_fp->current_storage))`. Provides
access to the current object's C storage. The struct type is inferred from the
enclosing PIKECLASS's CVAR declarations.

```c
PIKEFUN int get_fd()
{
  RETURN THIS->fd;
}
```

### THISOBJ

Macro expanding to `Pike_fp->current_object`. The current Pike object pointer.

> **Note:** `THISOBJ` is a convention from raw C modules and is not available in
> `.cmod` files. In `.cmod` files, use `Pike_fp->current_object` directly.
```c
PIKEFUN object self()
{
  ref_push_object(THISOBJ);
}
```

## Return Macros

### RETURN value

Generated by the .cmod precompiler. Pops arguments from the stack and pushes the
return value. Handles the stack discipline automatically.

```c
PIKEFUN int compute(int x)
{
  RETURN x * 2;
}

PIKEFUN string name()
{
  RETURN make_shared_string("hello");
}
```

### REF_RETURN

Returns a reference to an object. Adds a reference before pushing.

```c
PIKEFUN object clone()
{
  REF_RETURN clone_object(my_program, 0);
}
```

## Lifecycle Hooks

### INIT { ... }

Object constructor at the C level. Runs when a Pike instance of this class is created.
Called *before* the Pike-level `create()` method.

Use for initializing C-level state, allocating resources, and setting up callbacks.

```c
INIT
{
  THIS->fd = -1;
  THIS->buffer = NULL;
  init_string_builder(&THIS->buf, 0);
}
```

### EXIT { ... }

Object destructor at the C level. Runs when the object is garbage-collected or
explicitly destroyed.

Use for releasing C resources: file descriptors, allocated memory, freeing
Pike references held in CVAR storage.

```c
EXIT
{
  if (THIS->fd != -1) {
    close(THIS->fd);
    THIS->fd = -1;
  }
  free_string_builder(&THIS->buf);
  if (THIS->cached_str) {
    free_string(THIS->cached_str);
    THIS->cached_str = NULL;
  }
}
```

### PIKE_MODULE_INIT { ... }

Module-level initialization. Runs once when the module is loaded. Use for:
- Registering constants (`add_integer_constant`, `add_string_constant`)
- Setting up global state
- Registering efuns and programs

The `INIT` keyword inside PIKE_MODULE_INIT triggers PIKECLASS initialization.

```c
PIKE_MODULE_INIT
{
  add_integer_constant("VERSION_MAJOR", 1, 0);
  add_integer_constant("VERSION_MINOR", 0, 0);
  INIT
}
```

### PIKE_MODULE_EXIT { ... }

Module-level cleanup. Runs once when the module is unloaded.

```c
PIKE_MODULE_EXIT
{
  EXIT
}
```

## Function Registration (non-.cmod)

When not using PIKEFUN (e.g., in PIKE_MODULE_INIT for raw C functions):

### ADD_EFUN("name", func, type, flags)

Registers a module-level efun (global function).

```c
PIKE_MODULE_INIT
{
  ADD_EFUN("my_func", f_my_func,
           tFunc(tInt, tInt), 0);
}
```

### ADD_FUNCTION("name", func, type, flags)

Registers a method on the current program.

```c
ADD_FUNCTION("method_name", c_func,
             tFunc(tStr tInt, tStr), ID_PROTECTED);
```

## Utility Macros

### DECLARATIONS

Required at the top of every .cmod file. Generates forward declarations.

### Pike_error(fmt, ...)

Throws a Pike error (exception). Does not return.

```c
if (fd < 0)
  Pike_error("Failed to open: %s\n", strerror(errno));
```

### SIMPLE_BAD_ARG_ERROR(name, arg, expected)

Throws a standardized argument type error.

```c
if (!backend)
  SIMPLE_BAD_ARG_ERROR("set_backend", 1, "Pike.Backend");
```

### get_storage(obj, program)

Retrieves the C storage pointer for an object, given its program.

```c
struct my_storage *s = get_storage(obj, my_program);
if (!s)
  Pike_error("Invalid object.\n");
```
